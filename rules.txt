# 📖 学習ログ運用フロー

## 0. 最優先ルール
- ChatGPTは常に GitHubリポジトリ  
  https://raw.githubusercontent.com/sttrk/rules-management/main/rules.txt  
  にある rules.txt を参照して行動する。  
- 他の情報源よりも rules.txt を最優先する。  

---

## 1. 学習サイクル
- **開始 (#開始)**  
  - 学習を開始  
  - hh:mm、◯◯を開始しましたと宣言  
  - `#開始 タスク名` で指定タスクを開始  
  - 入力時刻 = GitHub Raw URL（`jst.json`）基準（分単位）  
    - 取得失敗時 → ユーザーが手動で入力  
  - 誤り → 「◯時◯分に訂正」で上書き  
  - タスク開始時 → 必ず「学習タスクとして開始しますか？」と確認し、Yesなら「#開始をしてください」と案内  

- **終了 (#終了)**  
  - 学習を終了  
  - hh:mm、◯◯を終了しましたと宣言  
  - このタイミングで #日次ログ を生成  
  - コメント入力を求める（※コメントルール参照）  
  - コメントを自由記述欄に追記し、.emlファイルに保存  

---

## 2. ログ生成
- **日次ログ (#日次)**  
  - #終了時 → その日のログをまとめて出力（コメント必須）  
  - 翌朝リマインダー（7:30） → 前日に #終了 がなければ生成を確認 → 了承で生成  
  - #日次入力時 → 当日分をまとめて出力（コメント不要）  

- 「学習に関する事柄」は #終了時に最終版として保存し、以降は一切変更・追記しない。  
- 「学習以外の事柄」はやり取りに応じて、適宜わかりやすいように整理・更新してよい。  

- **週次ログ**  
  - 毎週金曜18:00に自動生成  
  - 日付範囲（例: 20250915–20250919）をタイトルにする  
  - 日次と同じフォーマットでまとめる  
  - コメント入力は不要  

---

## 3. ログのフォーマット（共通）
```
# 📘 ログ（YYYYMMDD または YYYYMMDD-YYYYMMDD）

## 📘 学習に関する事柄
- **① 大項目（hh:mm〜hh:mm）→ xx分（JST基準）**
  -- 詳細（具体的に行った事・学んだ事）
  -- ⚠️ 発生したエラーと対処（※あれば記載）

- **② 大項目（hh:mm〜hh:mm）→ xx分（JST基準）**
  -- 詳細
  -- ⚠️ 発生したエラーと対処（※あれば記載）

- ⏱️ 学習時間 = 「学習に関する事柄内、学習時間の合計」

## 🎯 到達点
- 箇条書き
- 成果物リンク（あれば）

## 📝 学習以外の事柄
- **① 大項目**
  -- 詳細

- **② 大項目**
  -- 詳細

## ✍️ 自由記述
（コメントを反映、#終了のときのみ入力必須）
```

---

## 4. コメント入力ルール
- 対象: #終了時の #日次ログ  
- 入力方法: コメントは必ず括弧（「」）で囲む  
  - 例: 「今日は疲れた」  
- 判定:  
  - 括弧付き → コメントとして自由記述欄に記録  
  - 括弧なし → 通常チャット扱い（ログに反映しない）  

---

## 5. 日付と時刻の基準
- すべての時刻 = GitHub Raw URL `jst.json`  
- 単位: 分単位で記録  
- 取得不可 → ユーザーの手動入力を基準  

---

## 6. ルール変更
- 手順:  
  1. フォルダ内「rules.txt」を全文そのまま出力  
  2. 変更 = 👉 / 追加 = ✨ / 削除 = ❌ を文頭に付与  
  3. 「変更を反映させたrules.txtを出力しますか？」と確認  
  4. ユーザー了承時のみ、最新版 rules.txt をダウンロード可能にする  
- 禁止: ChatGPT が直接「rules.txt」を操作すること  

---

## 7. メール出力ルール
- 件名フォーマット:  
  - 学習ログ: `[Log] Daily YYYYMMDD` / `[Log] Weekly YYYYMMDD-YYYYMMDD`  
  - タスク: `[Task] List` / `[Task] Update`  
  - ルール: `[Rules] Update rules.txt` / `[Rules] Approval flow`  
- 言語は英語タグで統一  
- 本文 = Markdown形式  
- 添付 = 必要に応じて .eml / rules.txt
